Definition of Machine Learning by Tom Mitchell:
A computer program is said to learn from experience E with respect to some class of tasks T and performance measure P if its performance at tasks in T, as measured by P, improves with experience E.

Formal Definition of Deep Learning:
Deep learning is a particular kind of machine learning that achieves great power and flexibility by learning to represent the world as nested hierarchy of concepts, with each concept defined in relation to simpler concepts, and more abstract representations computed in terms of less abstract ones.

1. Data Dependencies
데이터가 충분히 많지 않을 때, 딥러닝은 기존의 머신러닝보다 상대적으로 큰 효과를 내지 못한다.
2. Hardware Dependencies
딥러닝 알고리즘은 본질적으로 대량의 행렬 곱셈을 필요로 하기 때문에 GPU의 사용이 필연적이다. GPU의 구축 목적 또한 이와 같다.
3. Feature Engineering
머신러닝에서 변수의 분류는 수작업이고 알고리즘의 성능이 이에 따라 좌우되는 반면, 딥러닝 알고리즘은 high level features를 학습하기 때문에 변수 추출 작업이 줄어든다. 예를 들어, Convolutional Neural Network에서 초기 레이어는 Raw data의 low level features를 학습하고 이를 토대로 다음 레이어에서 high level features를 학습한다.
4. Problem Solving Approach
전통적인 머신러닝 알고리즘의 문제 접근법은 단계별로 이루어지나 딥러닝 알고리즘은 end-to-end 방식으로 문제를 해결한다. 예로, grabcut과 같은 경계 탐지 알고리즘(bound box detection algorithm)은 먼저 가능한 객체를 탐색한 후에 이를 HOG가 있는 SVM (Support Vector Machine) 등의 객체 인식 알고리즘을 사용해 객체를 인식하는 반면 YOLO net은 객체의 이름과 위치를 한 번에 계산해낸다.
5. Execution Time
딥러닝은 train 시간이 오래 걸린다. Resnet의 경우 평균적으로 2주가 소요된다. 하지만 test의 경우, 보통 딥러닝에서 훨씬 빠르게 이루어진다. 다만 머신러닝의 test 속도는 알고리즘마다 편차가 크다. 가령 k-nearest neighbors의 경우 데이터 크기가 커질수록 테스트 시간이 길어진다.
6. Interpretability
의사결정나무(Decision tree learning), 선형회귀모형, Logistic regression 등의 경우 명확한 rule이 제공되기에 해석이 필요한 산업에서 이용할 수 있으나, 딥러닝 알고리즘에서는 이와 같은 활용이 어렵다.
